## 4. Architecturele keuzes (met motivatie)

1. **Closed status enum** (`active_discovery / planned / disabled /
   diagnostic`) — hard set, niet open. Maakt policy-engine
   reasoning deterministic en audit-baar.
2. **Exactly one active_discovery** — hard invariant
   geverifieerd at module import via `_validate_catalog`. Houdt de
   COL gefocust op één gecontroleerd experiment.
3. **Adjacent sidecars, geen frozen-contract uitbreiding** —
   `research_latest.json` + `strategy_matrix.csv` blijven ongerept;
   catalog + campaign metadata zijn aparte sidecars.
4. **`pullback_distance` als composite primitive** — geen indicator
   computation in strategy layer. Strategy declares 4 features;
   feature layer doet de werk. Layer purity behouden.
5. **`trend_pullback_v1` registered ALONGSIDE legacy
   `trend_pullback`** — geen breaking change op archived audit
   trails of bestaande presets die de legacy referenceren.
6. **Strategy failure taxonomy in apart module** — protects v3.11
   `rejection_taxonomy` contract; campaign-side taxonomy mag
   onafhankelijk evolueren.
7. **Default `require_hypothesis_status=()` + omit-when-empty** —
   v3.15.2 `campaign_templates_latest.v1.json` blijft byte-identical
   voor bestaande 3 baseline presets. Gepind door
   `test_v3_15_2_campaign_templates_byte_identity.py`.
8. **`live_eligible=False` hard-pinned** op alle nieuwe artifacts.

## 8. v3.15.4+ candidate directions

Natuurlijke vervolgen vanaf v3.15.3:

### Auto-status transitions (highest priority)

De catalog ships hand-curated. v3.15.4 kan een
`HypothesisStatusTransition` event introduceren waar de campaign
launcher op basis van accumulated evidence (5 cost_fragile in 7
ticks → cooldown; 10 no_baseline_edge in 30 ticks → negated;
etc.) een status mutation propose. Vereist:
- Nieuwe ledger event types in `campaign_evidence_ledger`
- Hypothesis-status state machine met legal transitions
- Schrijver naar `strategy_hypothesis_catalog_latest.v1.json` die
  initial-curated en runtime-mutated status separated houdt

### Multi-strategy hypothesis bundles

`bundle[0]` is een v3.15.3 simplificatie. v3.15.4+ kan een
explicit `hypothesis_strategies: tuple[str, ...]` field aan
`ResearchPreset` toevoegen zodat presets met meerdere strategies
expliciet welke hypotheses ze bedienen.

### atr_adaptive_trend executable

Tweede `active_discovery` candidate na trend_pullback_v1
voldoende validation evidence heeft (positief OF negatief — beide
geleerde signalen). Vereist:
- ATR-based feature primitives uitbreidingen
- Catalog status transition trend_pullback_v1 → cooldown +
  atr_adaptive_trend_v0 → active_discovery (preserve invariant)
- Nieuwe preset wired through templates

### v3.15.2 carry-overs (C1-C5)

Recommended priority C2 (build_campaign_id local TZ bug) → C1
(degenerate run misclassification) → C3 (R0 reclaim re-pick) →
C5 (lock-contention visibility) → C4 (GHA :latest tag staleness).

## 9. Waar te beginnen voor v3.15.4

Roadmap-volgorde blijft: research → candidate → regime → portfolio
→ paper → live. v3.15.3 maakt het mogelijk dat de COL autonoom
hypotheses selecteert/blokkeert. v3.15.4 kan beginnen met
**auto-status transitions** zodat de COL hypotheses zelf
deprioriteert/cooldownt op basis van accumulated evidence —
volgende stap richting echte autonomy.

Leidende filosofie blijft onveranderd: diagnostic-first, named
thresholds, gesloten taxonomies, additieve sidecars, geen
engine-contract uitbreiding, geen live surfaces.
