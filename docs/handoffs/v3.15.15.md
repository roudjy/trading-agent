# v3.15.15 — Vol Compression Breakout: 4h preset + template wiring + observability safeguards

Date: 2026-04-27
Branch: `feature/v3.15.15-vol-compression-templates`
VERSION bump: `3.15.14 → 3.15.15`

## Scope

Two coupled changes plus four observability-only safeguards:

1. **Add `vol_compression_breakout_crypto_4h` preset.** Mirrors the
   existing 1h variant but on 4h. Binds to the existing
   `volatility_compression_breakout_v0` hypothesis — no new hypothesis,
   no new strategy.
2. **Wire `vol_compression_breakout_crypto_*h` (1h + 4h) into
   `_HYPOTHESIS_AWARE_PRESETS`** so the v3.15.2 builder generates the
   standard five templates per preset. `CAMPAIGN_TEMPLATES` grows from
   20 → 30.
3. **Four observability-only safeguards** (no filtering, no policy
   mutation, decision logic stays in `campaign_policy.decide()`):
   - 4h `insufficient_trades` rate observations (tag emission only).
   - Static parameter coverage ratio per plan preset.
   - Per-preset throughput-regression check with a non-zero baseline
     floor.
   - Orthogonality validation across the `PRESETS` tuple.

After v3.15.15 the `crypto_exploratory_v1` sprint plan contains
**three** entries:

```

vol_compression_breakout_crypto_1h     (volatility_compression_breakout_v0, 1h)
vol_compression_breakout_crypto_4h     (volatility_compression_breakout_v0, 4h)
```

### `research/discovery_sprint.py`

Eight new public symbols, all observability-only:

- `compute_4h_insufficient_trades_observations(*, candidate_preset_names,
  campaign_registry, threshold=0.7, min_history=5) -> list[dict]`
  Per 4h candidate, computes
  `insufficient_trades_count / completed_runs`, emits a tag
  (`4h_insufficient_trades_high` / `..._ok` / `..._cold_start`),
  and returns observation records. **Never drops candidates.**
- `compute_parameter_coverage(*, plan, catalog=...,
  sample_limit=SCREENING_PARAM_SAMPLE_LIMIT) -> list[dict]`
  Static `min(sample_limit, grid_size) / grid_size` per plan preset.
  Sampling behavior unchanged.
- `compute_throughput_snapshot(*, campaign_registry, now_utc,
  window_days=THROUGHPUT_WINDOW_DAYS) -> ThroughputSnapshot`
  Per-preset spawn count + rate over `[now-window, now]`.
- `ensure_throughput_baseline(...)` — captures a baseline on first
  call, returns existing on subsequent calls (idempotent, never
  overwrites).
- `detect_throughput_regressions(*, baseline, current,
  drop_threshold=0.5, min_baseline_rate=THROUGHPUT_MIN_BASELINE_RATE)
  -> list[dict]`
  For each preset in baseline:
  `effective_baseline = max(baseline_rate, min_baseline_rate)`,
  `threshold_rate = effective_baseline * (1 - drop_threshold)`. The
  `min_baseline_rate=0.1` floor is the user-mandated zero-baseline
  guard — without it, a preset that was idle pre-deploy would either
  divide-by-zero or trip a false-positive on any post-deploy spawn.
- `check_preset_orthogonality(presets) -> list[dict]`
  Warns on any pair of presets sharing both `hypothesis_id` and
  `timeframe`. The 4h variant passes (timeframe differs from 1h).
- `build_safeguards_decision_payload(...)` +
  `write_safeguards_decision_artifact(...)`
  Aggregate the four signals + baseline/current snapshots into
  `research/discovery_sprints/sprint_safeguards_decision_latest.v1.json`.
  Top-level `observability_only: true`.

New constants:

| name | value | purpose |
|---|---|---|
| `SAFEGUARDS_DECISION_PATH` | `research/discovery_sprints/sprint_safeguards_decision_latest.v1.json` | the aggregate sidecar |
| `THROUGHPUT_BASELINE_PATH` | `research/discovery_sprints/throughput_baseline_v3_15_15.json` | the once-captured baseline |
| `SCREENING_PARAM_SAMPLE_LIMIT` | `3` | observability seam for v3.15.17 sampling rotation |
| `THROUGHPUT_WINDOW_DAYS` | `7` | rolling window for spawn-rate computation |
| `THROUGHPUT_DROP_THRESHOLD` | `0.5` | warn when current_rate drops below 50% of baseline |
| `THROUGHPUT_MIN_BASELINE_RATE` | `0.1` campaigns/day | zero-baseline floor |
| `INSUFFICIENT_TRADES_REASON_CODE` | `"insufficient_trades"` | reason_code matched against registry records |
| `INSUFFICIENT_TRADES_RATE_THRESHOLD` | `0.7` | tag-emission threshold (not a filter) |
| `INSUFFICIENT_TRADES_MIN_HISTORY` | `5` | cold-start guard (no high tag below this) |

None. v3.15.14 sprint state machine
(`active → completed | expired | canceled`) is unchanged. Sprint

## Known limitations / out of scope (deferred to v3.15.16+)

- v3.15.16: adaptive routing weighting on `degenerate_no_survivor`
  history.
- v3.15.17: stratified / rotated parameter sampling (the
  `parameter_coverage` observability seam shipped here is the read
  side; rotation is the write side).
- v3.15.18: extended sprint artifacts beyond the four landed here
  (failure_mode_distribution per preset, near-pass count, etc.).

The `vol_compression_breakout_crypto_4h` preset has no historical
runs at deploy time, so `compute_4h_insufficient_trades_observations`
will report `_cold_start` until at least
`INSUFFICIENT_TRADES_MIN_HISTORY=5` runs land. The 4h cooldown
defaults are inherited from the existing builder (24h daily_primary,
12h survivor_confirmation, etc.) — no override in v3.15.15.
