# v3.15.12 — Funnel Spawn Proposer (advisory shadow mode)

Date: 2026-04-27
Branch: `feature/v3.15.12-funnel-spawn-proposer`
VERSION bump: `3.15.11 → 3.15.12`

## Scope

Adds the first **forward-looking** module in the research intelligence
layer. Reads v3.15.9 screening_evidence + v3.15.11 advisory artifacts
+ v3.15.2 campaign_registry, emits proposed-but-not-spawned campaigns
under `research/campaigns/evidence/spawn_proposals_latest.v1.json`.

Hard positioning:

- `enforcement_state="advisory_only"` (top-level).
- `mode="shadow"` (top-level).
- `proposal_mode in {"normal", "diagnostic_only"}` (per-build behavior).
- Never mutates the queue, the registry, the ledger JSONL, or any
  frozen contract.
- `campaign_policy.decide()` remains unchanged — pinned by regression
  test that explicitly forbids `spawn_proposals`/`proposed_campaigns`/
  `advisory_decisions`/`stop_conditions` kwargs.

See `docs/funnel_spawn_proposer_design.md` for the full design + the
six operator-review hardenings.

## Modules

- `research/funnel_spawn_proposer.py` — pure builder + thin IO wrapper
  + append-only history JSONL helper. Eleven deterministic rules
  (R1–R11) + cross-cutting hardenings.
- `dashboard/api_research_intelligence.py` — extended with
  `/api/research/spawn-proposals` and a new `spawn_proposals` block
  in `/api/research/intelligence-summary`.
- `frontend/src/components/ResearchIntelligenceCard.tsx` — extended
  with proposal mode + count + suppressed zone count + optional
  human-review banner + top-3 proposals.

## Lifecycle integration

`research/run_research.py` calls `write_spawn_proposals_artifact(...)`
after `write_viability_artifact`. Wrapped in its own try/except +
`tracker_event`. Inputs are piped from the just-written v3.15.11
sidecars (evidence ledger, IG, stop conditions, dead zones, viability)
plus the v3.15.9 screening_evidence captured into the existing
defaulted local. Four payload variables (`ig_payload`, `stop_payload`,
`dz_payload`, `via_payload`) promoted to defaulted Optional locals so
the proposer can read them safely or degrade to `None` when their
owning try block failed.

## Cross-version dependencies

This release assumes:

- v3.15.5 outcome semantics
- v3.15.6 screening_phase propagation
- v3.15.7 exploratory criteria + `pass_kind`
- v3.15.8 `sampled_parameter_digest` (used as
  `parameter_grid_signature` in the proposal_fingerprint)
- v3.15.9 `screening_evidence_latest.v1.json`
- v3.15.10 funnel decisions (read-only signal source)
- v3.15.11 evidence_ledger / information_gain / stop_conditions /
  dead_zones / viability (full intelligence layer)

Frozen contracts (`research_latest.json`, `strategy_matrix.csv`,
`candidate_registry_latest.v1.json` schema) are NEVER touched.

## Out of scope (deferred)

- Phase 2 (v3.15.13 or v3.16.x): `proposal_evaluation.py` artifact
  comparing proposals to actuals, with operator-judgement adjustment
  of constants.
- Phase 3 (deferred): `campaign_policy.decide()` consumption behind a
  config flag with per-category caps.
- Dead-zone status decay inside `dead_zone_detection.py` itself
  (currently the proposer derives "time since last attempt in zone"
  from the evidence_ledger; the v3.15.11 dead_zones artifact does
  not yet carry this field).
- Strategy-family-aware lineage on confirmation proposals
  (currently uses parent_run_id only).
