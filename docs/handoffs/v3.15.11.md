# v3.15.11 — Research Intelligence Layer (advisory observability)

Date: 2026-04-27
Branch: `feature/v3.15.x-research-intelligence-layer`
VERSION bump: `3.15.10 → 3.15.11`

## Scope

Adds five deterministic, advisory-only sidecars on top of the v3.15.10
funnel completion. None of them modify policy, queue, registry, or
frozen contracts. See `docs/research_intelligence_layer.md` for the
full operator-facing description.

## Modules

- `research/research_evidence_ledger.py` — rolled-up snapshot of
  campaign events grouped by `(preset, hypothesis, family)`.
- `research/information_gain.py` — deterministic per-campaign score
  with named reasons. No ML.
- `research/stop_condition_engine.py` — advisory recommendations
  (`recommended_decision` + `enforcement_state="advisory_only"`).
- `research/dead_zone_detection.py` — `(asset × timeframe × family)`
  zone classifier. Signal only — never removes anything.
- `research/viability_metrics.py` — research viability verdict and
  cost-per-X metrics with safe denominators.

## Lifecycle integration

`research/run_research.py` calls the five `write_*_artifact` functions
in deterministic order after the v3.15.9 `screening_evidence` write.
Each is wrapped in its own try/except + tracker_event so partial
failure cannot mask the run's original outcome. `screening_evidence_payload`
is captured into a defaulted local in the v3.15.9 block so the
v3.15.11 block can read it safely or degrade to empty inputs when
v3.15.9 itself failed.

`campaign_policy.decide()` is **NOT** modified.

## API + dashboard

`dashboard/api_research_intelligence.py` adds:

- `GET /api/research/evidence-ledger`
- `GET /api/research/information-gain`
- `GET /api/research/stop-conditions`
- `GET /api/research/dead-zones`
- `GET /api/research/viability`
- `GET /api/research/intelligence-summary` (combined card-friendly view)

`frontend/src/components/ResearchIntelligenceCard.tsx` is a render-only
component with no business logic.
