# ADE-QRE-026 Autonomous ADE Work Admission, Execution Orchestration, and Research Throughput Engine

## Status

Admitted governed roadmap program for the bounded local orchestration layer that
connects canonical QRE next-action artifacts to typed work admission, existing
ADE-QRE-019 through ADE-QRE-025 capability routing, local validation, research
replay, preregistration, campaign execution, evidence ingestion, and portfolio
reprioritization.

`ADE-QRE-026R` is part of this same program and is not a separate PR lane. It
extends ADE-QRE-026 with one canonical operations configuration surface, simple
operator commands, typed monitoring/status artifacts, a daily intelligence
report, resumable control-state, and independence-aware throughput controls.

## Program scope

ADE-QRE-026 may:

- read canonical QRE next-action, readiness, campaign, hypothesis, strategy,
  primitive, portfolio, window-ledger, contradiction, and failure-memory
  artifacts;
- create typed next-action and work-item artifacts inside closed vocabularies;
- admit bounded autonomous work only inside approved work classes;
- route admitted work to repository-native ADE-QRE-019 through ADE-QRE-025
  functions and fixed validation commands;
- persist orchestration config, ledgers, status, monitoring, reports, work
  packages, and read models outside protected `research/**`;
- run bounded local remediation, replay, preregistration, and preregistered
  campaign execution only where existing research authority already permits it;
- enforce deterministic budgets, OOS conservation, resource claims, lock
  classes, concurrency limits, cycle limits, and kill-switch state;
- generate a deterministic daily research operations report, history, alerts,
  health classification, and next-24-hour plan.

ADE-QRE-026 may not:

- edit `.claude/**`;
- weaken protected `research/**` write denials or mutate frozen contracts;
- invoke arbitrary shell commands, arbitrary network access, arbitrary package
  installation, arbitrary remote agents, arbitrary GitHub execution, or hidden
  self-modifying code;
- fabricate external execution, fabricated evidence, fabricated data, invented
  identities, invented windows, or invented independent OOS;
- reuse consumed OOS as independent evidence, lower scientific gates, mutate a
  strategy after evidence, or tune against OOS;
- grant paper, shadow, live, broker, execution, risk, capital-allocation, or
  deployment authority.

## Definition of done

ADE-QRE-026 is complete only when:

- one canonical resolved research portfolio exists and gives every tracked
  strategy exactly one current stage and one primary next action;
- typed next-action, work-admission, routing, validation, status, alert, and
  closeout vocabularies are enforced in code and tests;
- repository-native capability adapters invoke only allowlisted functions or
  fixed validation commands;
- orchestration artifacts, ledgers, work packages, reports, and read models
  are written only to approved generated-research surfaces;
- one typed operations configuration controls capacities, budgets, diversity,
  concurrency, stop conditions, and daily reporting without source edits;
- the pre-OOS conservation gate blocks low-information OOS consumption before
  execution unless the frozen policy proves adequate expected OOS capacity;
- the bounded local loop can ingest the ADE-QRE-025 next action
  `launch_data_oos_capacity_expansion`, admit a typed work item, execute safe
  repository-native work, replay affected downstream research, and produce the
  exact next autonomous action;
- daily JSON and Markdown operations reports, compact status artifacts, trend
  history, and frontend-ready monitoring schemas are generated deterministically
  and remain resumable after interruption;
- the implementation preserves `.claude/**`, protected empirical
  `research/**`, frozen contracts, and the no-trading boundary.
