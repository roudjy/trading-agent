# ADE-QRE-021 Automated Bounded Primitive Expansion and Strategy Retest Program

## Status

Active governed follow-on program after ADE-QRE-020.

ADE-QRE-021 admits bounded, deterministic, research-only primitive expansion
through isolated generated-primitive surfaces outside `research/**`.

## Authority

ADE-QRE-021 may:

- ingest authoritative machine-readable primitive-extension requests from
  ADE-QRE-020 or an equivalent governed source;
- validate bounded primitive-extension requests against closed contracts;
- compile closed primitive specifications;
- generate deterministic primitive implementations and deterministic tests;
- run static safety, architecture, and isolated sandbox validation;
- automatically admit validated generated primitives into an isolated generated
  primitive registry;
- compose manual and generated primitives through one resolved primitive
  catalog;
- automatically recompile affected theses through ADE-QRE-019;
- automatically trigger ADE-QRE-019 strategy regeneration, validation,
  research-only registration, preset generation, null-control specification,
  lineage materialization, and campaign-readiness reevaluation.

ADE-QRE-021 may not:

- generate arbitrary feature or indicator libraries without an authoritative
  bounded request;
- bypass ADE-QRE-019 strategy gates;
- execute campaigns;
- treat primitive or sandbox validation as market, alpha, OOS, or null-control
  evidence;
- grant paper, shadow, live, broker, risk, execution, or capital authority;
- write to protected `research/**` surfaces;
- modify `.claude/**`.

## Definition of Done

ADE-QRE-021 is complete only when:

- isolated generated-primitive surfaces exist outside `research/**`;
- the primitive-extension request contract, primitive specification, generated
  primitive implementation, generated tests, validation artifacts, generated
  primitive registry, and resolved primitive catalog are persisted;
- the requested primitive `cross_sectional_rank` is either automatically
  registered for research-only use or quarantined with exact blockers;
- thesis `qhc_51bc7a5c7b3f64ba` is automatically recompiled and sent through the
  existing ADE-QRE-019 pipeline when the primitive is resolver-visible;
- campaign readiness is reevaluated without campaign execution;
- tests, governance lint, architecture tests, and architecture import scan are
  green;
- frozen contracts and protected execution surfaces remain unchanged.
