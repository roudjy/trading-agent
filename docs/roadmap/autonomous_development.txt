# 1\. Core correction

The project now has two distinct top-level tracks.

## 1.1 Autonomous Development Track

Purpose:

```text
Make Claude/agent-driven development safe, deterministic, observable, approval-gated, CI-governed, and operator-controllable.
```

This track governs how development work is proposed, classified, approved, tested, merged, deployed, and observed.

It is meta-infrastructure for the development process.

It is **not** QRE trading, research, paper, shadow, broker, or execution-engine product functionality.

## 1.2 QRE Feature Build Track

Purpose:

```text
Build the actual Quant Research Engine product roadmap.
```

This track is defined by the original Roadmap v6, which the operator maintains locally.

After the Autonomous Development Track is complete, the project returns to the QRE Feature Build Track at:

```text

```

The next QRE feature sequence after that remains the Roadmap v6 product sequence:


Do not reconstruct this product roadmap inside this document. Use the operator-provided Roadmap v6 as the product-roadmap source of truth.

# 2\. Terminology guardrail

The word `execution` has caused ambiguity and must be disambiguated.

## 2.1 Agent execution authority

Means:

```text
May Claude/the agent perform this repository-development action automatically,
should it require operator approval,
or is it permanently denied?
```

This belongs to the Autonomous Development Track.

Examples:

```text
file edits
branch creation
commit creation
PR creation
CI checks
roadmap edits
governance policy edits
frozen contract changes
live-path changes
approval-inbox mutations
```

## 2.2 Trading/research execution

Means:

```text
May the QRE system execute strategies, campaigns, orders, paper trades,
shadow trades, broker interactions, risk decisions, or capital allocation?
```

This belongs to the QRE Feature Build Track and later semantic maturity layers.

Agent Execution Authority does **not** grant trading execution authority.
