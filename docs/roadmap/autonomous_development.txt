# 2\. Terminology guardrail

The word `execution` has caused ambiguity and must be disambiguated.

## 2.1 Agent execution authority

Means:

```text
May Claude/the agent perform this repository-development action automatically,
should it require operator approval,
or is it permanently denied?
```

This belongs to the Autonomous Development Track.

Examples:

```text
file edits
branch creation
commit creation
PR creation
CI checks
roadmap edits
governance policy edits
frozen contract changes
live-path changes
approval-inbox mutations
```

## 2.2 Trading/research execution

Means:

```text
May the QRE system execute strategies, campaigns, orders, paper trades,
shadow trades, broker interactions, risk decisions, or capital allocation?
```

This belongs to the QRE Feature Build Track and later semantic maturity layers.

Agent Execution Authority does **not** grant trading execution authority.
