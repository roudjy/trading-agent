# v3.15.15.12 — Claude Agent Governance & Safety Layer

## Status

In progress (initial implementation pass on branch
`feat/v3.15.15.12-agent-governance`).

## Goal

Build the safety perimeter required *before* Claude agents are given any
autonomy on roadmap items. Strict ordering: secrets first, then CI hardening,
then hooks, then agents, then audit/provenance, then ADR-015, then dry-runs,
then release gate.

The governance layer is the architectural counterpart of ADR-014's truth-
authority pluralism: where ADR-014 settles **what is canonical**, ADR-015
settles **who is allowed to change it and how**.

## Stop conditions for the version

The version closes when:

1. Branch protection on `main` is enabled per
   [`branch_protection_checklist.md`](../governance/branch_protection_checklist.md).
2. The first release-gate report exists for this PR.
3. The first rollback drill is logged in
   `docs/governance/rollback_drills/<date>.md`.
4. The first monthly SHA-pin review is logged in
   `docs/governance/sha_pin_reviews/<YYYY-MM>.md`.
5. CHANGELOG entry and `VERSION` bump are merged via release-gate-recommended PR.

Items 1–4 are operator tasks scheduled outside this PR. The implementation
PR itself is scoped to repository-side work only.
