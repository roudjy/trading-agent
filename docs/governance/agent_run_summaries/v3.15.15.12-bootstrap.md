- `outcome=blocked_by_hook` count: **6** (live-author demonstrations of the hook layer):
  - by reason:
    - `live_connector_create` — blocked an update to `deny_live_connector.py` itself because the new content matched its own regex (resolved by tightening the content scan to `.py` files only)
    - `live_connector_create` — blocked governance docs that *describe* live patterns (resolved by adding markdown-skip)
    - `test_weakening` — blocked the test file that tests the test-weakening hook itself (resolved via `governance-bootstrap` Python script)
    - `dangerous_bash` matched `edit_settings_local` — blocked a commit message containing the literal `.claude/settings.local.json` (resolved by sanitizing the message)
    - `dangerous_bash` matched `python_dash_c` — blocked a commit message that quoted `python -c` (resolved)
    - `read_config_yaml` — blocked a commit message that used `git commit -m "$(cat <<EOF ... config/config.yaml ... EOF)"` heredoc form (resolved by switching to inline `-m`)

Each block was a correct positive given the hook regex semantics. No false-positives slipped through to the merge gate.

## Notes

This session implemented the v3.15.15.12 governance layer end-to-end across
12 commits, three review rounds, and one targeted blocker-fix. The most
load-bearing observation is that the hook layer enforced its own
constraints on its own author six times during implementation, and each
block was resolved through the documented `governance-bootstrap` path
(plain Python file I/O via `.tmp/` or `scripts/_bootstrap_*.py` helpers
that were removed before commit).

Three known gaps are deliberately deferred and tracked in
`docs/backlog/agent_backlog.md`:

- `find -name 'config.yaml'` (basename without path prefix) — regex
  intentionally tight to avoid false positives.
- Whole-ledger external attestation (Merkle root or signed checkpoint) —
  out of scope for this PR.
- GHCR-attached attestation vs the artifact-only provenance — backlog
  `AB-0002`.

External credential rotation is **deferred** while live trading is disabled
and the Polymarket wallet has no funds, per `manual_blockers.md`. Re-evaluate
before any v3.16+ live work.
