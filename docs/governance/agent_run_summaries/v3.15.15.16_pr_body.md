## Summary

Adds the **Autonomous Workloop Controller** in **Local Planning
Mode**. New module `reporting.autonomous_workloop` classifies open
PRs / Dependabot bumps / roadmap items, runs local gates, and emits a
paired digest (markdown + JSON). It does **not** merge anything, push
to `main`, call any GitHub API, install `gh`, or recommend
`safe_to_merge` in this release.

Workloop maturity: this release reaches Levels B + C. Levels D–G
(GitHub-backed PR awareness, safe PR execution, dashboard execute-
safe controls, scheduled runtime, safe automerge) are scoped to
v3.15.15.17 → .23 in the new roadmap doc bundled with this PR.

## Why

The operator wants to leave the manual ChatGPT ↔ Claude ↔ GitHub
routing loop. v3.15.15.16 turns "decide what to merge" into "click
confirm on already-vetted decisions" by surfacing the queue + risk
classification + frozen-contract integrity + audit-chain status in a
single read.

## What this is NOT

- Not full PR automation. `gh` / API not available; `checks_status`
  is `not_available` for every branch in this release.
- Not a writer change. `.claude/**` is untouched.
- Not a merge surface. `merges_performed` is `0` invariantly.
- Not a roadmap executor. Roadmap selection is recommendation-only;
  the controller never opens a roadmap implementation branch.
- Not source-of-truth attribution (that's still ADR-016).

## Operator commands

```sh
python -m reporting.autonomous_workloop                          # default: dry-run, JSON to stdout
python -m reporting.autonomous_workloop --mode execute-safe      # write digest pair
python -m reporting.autonomous_workloop --mode continuous --max-cycles 5
python -m reporting.autonomous_workloop --mode digest

cat docs/governance/autonomous_workloop/latest.md
cat logs/autonomous_workloop/latest.json | jq .next_recommended_item
```

## Hard guarantees (encoded as tests)

1. `safe_to_merge` is unreachable in this release — six fixture
   classes (clean / protected / frozen / live / conflict / mixed) all
   land somewhere else.
2. `merges_performed == 0` invariant.
3. Push allowlist: only the current release branch
   (`fix/v3.15.15.16-autonomous-workloop-controller`); `main` /
   `dependabot/**` / unrelated branches are explicitly denied.
4. `execute-safe` defense-in-depth: every no-touch / frozen / live
   path is denied at the controller level (the deny_no_touch hook
   remains the canonical enforcement above this).
5. Roadmap recommendation-only: no autonomous branch creation.
6. JSON artifact carries every required key; `frontend_control_state`
   is present (forward-compat anchor for v3.15.15.17).
7. The 10 final-report statements appear verbatim in every digest.
8. Frozen contracts byte-identical before/after every snapshot.
9. `unknown` / `not_available` is never reported as `ok`.
