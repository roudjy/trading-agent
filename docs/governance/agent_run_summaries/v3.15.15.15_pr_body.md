## Summary

Adds a read-only inferred-subagent-attribution surface on top of the
agent audit ledger. Adds a `--view attribution` to the existing audit
summary CLI. Drafts the writer-level fix (ADR-016) at
`docs/governance/proposals/` for the operator to apply via a separate
governance-bootstrap PR.

This release is the bridge between today's actor-grain ledger
(`claude:audit_emit` / `claude:hook` / `claude:precompact_preserve`)
and per-subagent attribution. The bridge is **convenience-only, not
source-of-truth**.

## Why

The runtime ledger captures actor at the hook layer, not at the
subagent layer. Without modifying `.claude/hooks/audit_emit.py` (no-
touch by Doctrine 7 + 12, mechanically un-bypassable from an
autonomous session), the only way to surface subagent attribution
today is to reconstruct it post-hoc from already-committed run
summaries. This PR adds that surface — and labels it conservatively.

## What this is NOT

- Not a writer change. `.claude/hooks/**` is untouched.
- Not source-of-truth. Every row carries `confidence` and a
  `warning`; low-confidence rows must never be displayed as fact.
- Not a hook bypass. Any hook test must keep passing; governance-lint
  green; chain intact.
- Not auto-applied. The 15b ADR-016 proposal is a separate PR the
  operator opens by hand if writer-level attribution is desired.

## Confidence rules (round-3 tightened)

`confidence: high` requires *explicit source evidence*. Tool-count
matching is supporting only. Exactly one of:

1. Run summary has explicit per-event sequence_id mapping.
2. Transcript metadata has explicit per-sequence_id subagent id.
3. Run summary lists exactly **one** subagent for the entire session
   AND has no competing/conflicting evidence.

Otherwise: `low` (partial / ambiguous evidence) or `unknown`
(missing / malformed / conflicting).
